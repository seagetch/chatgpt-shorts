# Shorts Reader Maintainer

## Overview

This skill captures the repo-specific rules for the shared short-story reader. Use it when changing reader behavior, regenerating pages, or checking how `doc.md` and `title.jpg` are wired into the published site.

## When To Use

Use this skill when the task involves any of these files or behaviors:

- `book.html`
- `generate_reader.py`
- root `index.html`
- any work folder `index.html`
- pagination, spread layout, swipe or key navigation, font loading, social card metadata
- regenerating per-work pages after changing generator logic

## Workflow

1. Read [references/repo-layout.md](references/repo-layout.md) for the repo map and source-of-truth files.
2. If the change touches the shared reader runtime, edit `book.html`.
3. If the change touches generated wrappers or the root catalog, edit `generate_reader.py`.
4. After changing `generate_reader.py`, run:
   - `python -m py_compile generate_reader.py`
   - `python generate_reader.py`
5. Verify the reader over HTTP, not `file://`.
   - Local check: `python -m http.server`
   - Then open `/book.html?book=<folder-name>`

## Repo Rules

- `doc.md` is the source of truth for each story.
- `book.html` loads `doc.md` and `title.jpg` at runtime via HTTP using the `book` query parameter.
- Each work folder `index.html` is a generated thin redirect to `../book.html?book=<folder>`.
- Root `index.html` is generated by `generate_reader.py`.
- `title.jpg` is generated from `title.png` by `generate_reader.py`.
- Do not rely on `file://` for reader behavior; runtime markdown loading is HTTP-only by design.
- Do not commit or push unless the user explicitly asks.

## Notes

- If pagination or layout breaks only after generator changes, regenerate all pages before judging the result.
- If runtime loading fails, check the folder name passed in `?book=` and confirm `doc.md` and `title.jpg` exist in that folder.
